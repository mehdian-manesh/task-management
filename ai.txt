## System Overview
This is a full-stack task management application designed to track working hours and task reports. The system uses Django REST Framework for the backend and React for the frontend.

## Core Components

### Backend (Django + DRF)
- Custom User model
- JWT Authentication
- Nested routing with drf-nested-routers
- PostgreSQL database
- Timezone: Asia/Tehran

### Frontend (React)
- Material-UI with RTL support
- Redux Toolkit for state management
- JWT authentication
- Persian calendar support
- Responsive design

## Models

### User
- Standard Django user with is_staff flag for admin privileges

### WorkingDay
- user: ForeignKey to User
- check_in: DateTimeField (auto_now_add)
- check_out: DateTimeField (nullable)
- is_on_leave: BooleanField

### Report
- working_day: ForeignKey to WorkingDay
- task: ForeignKey to Task
- result: CharField (choices: ongoing, success, postponed, failed, cancelled)
- comment: TextField (max_length=1000)
- start_time: DateTimeField
- end_time: DateTimeField

### Task
- name: CharField
- description: TextField
- color: CharField
- project: ForeignKey to Project (optional)
- assigned_users: ManyToManyField to User
- start_date: DateField
- deadline: DateField
- estimated_hours: IntegerField
- phase: IntegerField
- status: CharField (choices: postpone, backlog, todo, doing, test, done, archive)
- is_draft: BooleanField

### Project
- name: CharField
- description: TextField
- color: CharField
- assigned_users: ManyToManyField to User
- start_date: DateField
- deadline: DateField
- estimated_hours: IntegerField
- status: CharField (same choices as Task)

### Feedback
- user: ForeignKey to User
- type: CharField (choices: criticism, suggestion, question)
- description: TextField
- created_at: DateTimeField

## API Endpoints

### Authentication
- POST /login/ - Login with username/password
- POST /logout/ - Logout user
- POST /token/refresh/ - Refresh JWT token

### Working Days
- GET /working-days/ - List working days
- POST /working-days/ - Check in
- POST /working-days/{id}/check_out/ - Check out
- POST /working-days/{id}/leave/ - Mark as leave
- GET /working-days/{id}/reports/ - List reports for working day
- POST /working-days/{id}/reports/ - Create report

### Tasks
- GET /tasks/ - List tasks
- POST /tasks/ - Create task
- GET /tasks/{id}/ - Get task details
- PATCH /tasks/{id}/ - Update task
- DELETE /tasks/{id}/ - Delete task

### Projects
- GET /projects/ - List projects
- POST /projects/ - Create project (admin only)
- GET /projects/{id}/ - Get project details
- PATCH /projects/{id}/ - Update project (admin only)
- DELETE /projects/{id}/ - Delete project (admin only)

### Feedbacks
- GET /feedbacks/ - List feedbacks
- POST /feedbacks/ - Create feedback
- PATCH /feedbacks/{id}/ - Update feedback
- DELETE /feedbacks/{id}/ - Delete feedback

## Development Environment

### Docker Setup
- Development: `docker-compose -f docker-compose.dev.yml up --build`
- Production: `docker-compose up --build`

### Services
1. Backend (Django):
   - Port: 8000
   - Hot reload enabled in dev

2. Frontend (React):
   - Port: 3000
   - Hot reload enabled in dev
   - RTL and Persian localization

3. Caddy (Reverse Proxy):
   - Ports: 80, 443
   - Auto SSL in production
   - Dev uses self-signed certificates

4. PostgreSQL:
   - Port: 5432
   - Persisted volume storage

## Key Features

### Task Workflow
1. Regular users can create draft tasks
2. Admin approves tasks, moving them from draft
3. Tasks can be associated with projects
4. Task status updates based on report results

### Working Day Management
1. Check-in creates a working day
2. Multiple reports can be added during the day
3. Check-out closes the working day
4. Leave can be marked without check-out

### Authorization Rules
1. Regular users:
   - View assigned projects
   - Create draft tasks
   - Manage own working days and reports
   - Manage own feedback

2. Admins:
   - Full project CRUD
   - Task approval
   - View all feedback
   - User assignment

## Conventions
1. All dates use Persian calendar (Shamsi)
2. Times are in Tehran timezone
3. All text content is in Persian
4. RTL layout throughout the application
